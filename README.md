# Software Design

Este repositório contém estudos, anotações e exemplos práticos sobre **Software Design**, com foco na construção de código limpo, modular, reutilizável e de fácil manutenção.

Software Design trata da forma como o código é estruturado **no nível de classes e componentes**, aplicando princípios e padrões que ajudam a reduzir acoplamento, aumentar coesão e melhorar a qualidade do software.

## Tópicos abordados

- Programação Orientada a Objetos
- Princípios SOLID
- Clean Code
- Refatoração
- Injeção de Dependência
- Boas práticas de design
- Código testável

## Design Patterns estudados

- Strategy
- Factory
- Builder
- Singleton
- Observer
- Proxy
- Template Method
- Chain of Responsibility

## Objetivos

- Melhorar a qualidade e legibilidade do código
- Aprender a estruturar classes e módulos corretamente
- Aplicar padrões de projeto amplamente utilizados na indústria
- Desenvolver software mais flexível, escalável e testável

## Tecnologias utilizadas

Os exemplos podem incluir:

- Java
- Spring Boot
- Testes unitários
- Maven / Gradle

## Propósito

Este repositório funciona como uma **base de conhecimento pessoal e material de estudo** sobre boas práticas de design aplicadas ao desenvolvimento de software.
